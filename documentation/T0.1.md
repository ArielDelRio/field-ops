# T0.1 - Inicializacion del proyecto Expo

## Objetivo

Crear el proyecto base con Expo y dejar listo el flujo minimo para desarrollo y validacion inicial.

## Que hubo que hacer

- Se creo la app con `pnpm create expo-app --template default@sdk-56`.
- La app quedo dentro de `field-ops/app`.
- Se reviso la estructura inicial generada por Expo.
- Se ejecuto `reset-project` para eliminar el contenido demo y dejar una base minima con Expo Router.
- Se dejo un layout raiz simple con `Stack`.
- Se dejo una pantalla inicial minima en `src/app/index.tsx`.
- Se agrego el script `typecheck` con `tsc --noEmit`.
- Se configuro Jest con `jest-expo`.
- Se agrego el script `test`.
- Se agrego un test basico de renderizado con `@testing-library/react-native`.
- Se actualizo `tsconfig.json` para incluir tipos de Jest.
- Se valido la app en iOS Simulator.

## Resultado final

- La app arranca con `start`.
- El proyecto usa TypeScript estricto.
- `typecheck` pasa sin errores.
- `test` queda configurado y ejecuta una prueba simple.
- La base de `src` quedo limpia para empezar a desarrollar desde cero.

## Validaciones realizadas

- `pnpm -C field-ops/app run typecheck`
- `pnpm -C field-ops/app test`
- Ejecucion en iOS Simulator

## Lo aprendido

- `typecheck` sirve para validar tipos sin depender de levantar Expo.
- `start` levanta el entorno de desarrollo, pero no reemplaza una validacion dedicada de TypeScript.
- `Jest` ejecuta y organiza las pruebas.
- `React Native Testing Library` renderiza componentes y permite consultar la UI.
- En este setup, `render(...)` debe resolverse de forma asincrona en el test actual.

## Decisiones tomadas

- No agregar snapshot tests en `T0.1` porque todavia no hay componentes visuales estables que justifiquen congelar salida de UI.
- No avanzar todavia en tema, features o navegacion compleja porque esta tarea solo buscaba dejar la base lista.
- Se elimino el script `reset-project` del `package.json` despues de usarlo, porque el archivo asociado ya no existe.
